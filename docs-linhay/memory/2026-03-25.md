# 2026-03-25

## SKIntelligence issue #3 修复记录
- 决策：`SKIMCPClient.callTool` 的 `Tool.Content.text` 解析改为版本兼容策略，不再依赖固定关联值签名。
- 原因：上游 MCP SDK 在新版本中可能将 `.text` 从单值扩展为多关联值（如 `text/annotations/_meta`），旧写法会在上层工程编译时报类型不匹配。
- 实施：新增 `SKIMCPClient.extractText(_:)` 与 `extractTextPayload(_:)`，优先处理 `String`，并通过 tuple 反射兜底提取 `text` 字段或首个字符串值。
- 验证：新增 `SKIMCPClientTextExtractionTests` 覆盖字符串 payload、tuple-like payload、非文本 payload；并回归 `SKIMCPIntegrationTests`。
- 风险：tuple 反射依赖 text 字段命名或首字段语义，若上游未来再次破坏该约定，需要补充解析分支。
