## MLX 接入继续推进（E2E 运行时约束收敛）

- 发现问题：
  - 开启 `RUN_MLX_E2E_TESTS=1` 后，运行时可能直接报错并退出：
    - `Failed to load the default metallib`
  - 结论：属于 MLX Metal 运行时资源（`*.metallib`）缺失，不是适配层映射逻辑错误。
- 处理决策：
  - `SKIMLXClient` 目标依赖补齐 `MLXLLM`（确保模型工厂注册链路被链接）。
  - E2E 用例增加前置探测：若 `.build` 下未发现 `*.metallib`，直接 `XCTSkip` 并给出明确原因，避免底层崩溃。
  - E2E 超时改为可配置：`MLX_E2E_REQUEST_TIMEOUT_SECONDS`（默认 `180`）。
- 验证结果：
  - `RUN_MLX_E2E_TESTS=1 MLX_E2E_MODEL_ID=... swift test --filter MLXClientDeterminismE2ETests`
  - 结果：在当前环境稳定 `skip`（无崩溃），提示 `No Metal library (*.metallib) found in build products`。
  - `swift test --filter MLXClientTests` 回归通过（21 tests，1 skip，0 failure）。

## MLX 接入继续推进（真实 E2E 可执行化）

- 关键结论：
  - `mlx-swift` 官方说明：`swiftpm` 命令行无法构建 Metal shaders，需通过 `xcodebuild`/Xcode 产出 `default.metallib`。
- 落地动作：
  - 新增脚本：`scripts/mlx_e2e_prepare.sh`
    - 若找不到 `default.metallib`，自动执行：
      - `xcodebuild build -project .build/checkouts/mlx-swift/xcode/MLX.xcodeproj -scheme Cmlx -destination 'platform=macOS' -configuration Debug`
    - 将产物复制到仓库根目录 `default.metallib`；
    - 可选择直接执行 E2E（`--run`）。
  - 为避免误提交二进制，`.gitignore` 新增 `default.metallib`。
  - E2E 测试增强：
    - metallib 预检路径扩展到：仓库根目录、`.build`、`DYLD_FRAMEWORK_PATH`、`MLX_E2E_METALLIB_DIR`；
    - 默认 `MLX_E2E_TEMPERATURE=0`，提高同 seed 的稳定性。
- 验证结果：
  - `scripts/mlx_e2e_prepare.sh --run --model-id mlx-community/Qwen2.5-0.5B-4bit --timeout-seconds 120 --temperature 0` 可直接跑通；
  - `swift test --filter MLXClientDeterminismE2ETests` 在上述条件下通过；
  - `swift test --filter MLXClientTests` 全量回归通过（23 tests，1 skip，0 failure）。
