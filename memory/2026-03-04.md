## MLX 多模态流式输入图片透传修复

- 背景：VLM 流式路径中，最后一轮 prompt 的图片在 `streamSession` 调用时被固定置空（`images: []`），导致图片问答退化为纯文本。
- 决策：把最后一条消息中的 `images` 与 `prompt` 一起作为流式输入透传给 `session.streamDetails`。
- 代码改动：
  - `Sources/SKIMLXClient/MLXClient.swift`
    - 新增 `prepareStreamingInput(from:)`，统一提取 `history/prompt/images`。
    - `streamGenerate` 改为使用 `streamInput.images`。
    - `streamSession` 新增 `images` 参数并传入 `streamDetails(images:)`。
  - `Tests/SKIMLXClientTests/MLXClientTests.swift`
    - 新增 `testPrepareStreamingInputPreservesLatestUserImages`，覆盖“最后一轮 user 图片必须被保留”。

  2. `swift test --filter MLXClientTests` 通过（E2E 开关测试按环境变量跳过）。

## MLX 流式输入尾消息角色语义修复

- 背景：`prepareStreamingInput` 之前无条件剥离最后一条消息作为 prompt，导致尾消息为 `.tool` 等非 user 时被错误降级为普通文本 prompt。
- 决策：仅当最后一条消息 `role == .user` 时，才把它作为 `prompt/images` 抽离；否则保留完整 history，并使用空 prompt。
- 代码改动：

## MLX stop 语义与 ACP 多会话复用修复

- 背景：
  - stop 序列在 `MLXRequestOptions.normalizedStopSequences` 中被 `trim`，会破坏调用方有意传入的空白边界（如 `"\n\n"`、`" END"`）。
  - `acp serve --model-provider mlx` 下，`sessionFactory` 每次创建会话都新建 `MLXClient`，导致同模型重复加载。
- 决策：
  - stop 序列仅过滤真正空串，不做任何首尾空白裁剪，保持请求语义精确一致。
  - 在 `ACPServeCommand.run` 内按 serve 生命周期构建一次 `sharedMLXClient`，所有新建/fork 会话复用该实例。
- 代码改动：
  - `Sources/SKIMLXClient/MLXClient.swift`
    - `normalizedStopSequences` 从 `trim + filter` 改为 `filter { !$0.isEmpty }`。
  - `Sources/SKICLI/CLI.swift`
    - 新增 `sharedMLXClient` 并在 `.mlx` 的 `sessionFactory` 分支复用。
  - `Tests/SKIMLXClientTests/MLXClientTests.swift`
    - 新增 `testRespondPreservesExactStopSequencesIncludingWhitespace`。
- 验证：
  1. `swift test --filter MLXClientTests` 通过（含新 stop 用例，E2E 开关项按环境变量跳过）。
  2. `swift test --filter SKICLIProcessTests` 通过（98/98）。

## Linux 检查镜像瘦身方案

- 问题：`skintelligence-linux-check:latest` 体积偏大，当前 Dockerfile 使用 `swiftlang/swift:nightly-6.2-jammy` 单阶段构建，最终镜像携带完整 Swift 工具链。
- 决策：改为多阶段构建：
  - `build-check` 阶段：`swift:6.2-noble` 执行 `swift package resolve` + `swift build -c release`
  - 最终阶段：`ubuntu:24.04`，仅复制 `build-check.ok` 标记文件
- 效果：发布到仓库的最终镜像可降到 Ubuntu 基础层量级（约 28 MiB），不再携带 1GiB+ Swift 工具链层。
- 说明：首次构建仍需拉取 builder 工具链层（用于编译校验），但 registry 中的最终 `latest` 显著变小。
