## 直接安装脚本（源码构建）

- 新增脚本：`scripts/install_ski_source.sh`
  - 支持从本地源码目录或 GitHub 仓库直接构建并安装到指定前缀。
  - 默认安装到 `~/.local/bin/ski`，可通过 `--prefix` 覆盖。
- 版本可读性：
  - 安装脚本会写入 sidecar 文件 `ski.version`。
  - `SKICLIVersion` 新增 sidecar 读取逻辑，保证源码安装后 `ski --version` 可显示稳定版本而非 `dev`。
- 验证：
  1. `swift test --filter CLIVersionTests --filter CLIDistributionScriptsTests` 通过。
  2. 线测 `scripts/install_ski_source.sh --source-dir "$PWD" --prefix /tmp/ski-local --config debug` 后，`/tmp/ski-local/bin/ski --version` 输出 `2.0.3`。

## Gemini 对 TUI 的评估

- 报告位置：`docs-dev/dev/TUI-Evaluation-2026-03-06.md`
- 关键结论：
  1. 当前 TUI 实现适合作为内部工具和预览版继续推进。
  2. 优点集中在 UTF-8/CJK 支持、事件循环结构、终端恢复和轻量差异渲染。
  3. 主要短板是 Darwin 平台绑定、缺失 `SIGWINCH` resize 处理，以及渲染边界测试不足。
- 后续优先级：
  1. 先补窗口 resize 响应。
  2. 再补列宽/emoji/组合字符渲染测试。
  3. 最后再考虑 transcript 浏览和 Linux 适配。
