## Homebrew 二进制公式切换（避免安装时下载 Swift）

- 背景：用户安装 `linhay/tap/ski` 时触发 `swift (6.2.4)` 下载，说明 tap 公式仍为源码构建模式。
- 处理：
  1. 先补红灯测试：`Formula/ski.rb` 必须包含 `ski-macos-*.tar.gz`，且不得出现 `depends_on "swift" => :build` 与 `swift build`。
  2. 打包并上传 release 资产到 `2.0.0`：
     - `ski-macos-arm64.tar.gz` / `.sha256`
     - `ski-macos-x86_64.tar.gz` / `.sha256`
  3. 用 `scripts/generate_homebrew_formula.sh` 生成二进制公式并覆盖 `Formula/ski.rb`。
  4. 通过 `scripts/sync_homebrew_tap.sh` 推送到 `linhay/homebrew-tap`（commit `110034a`）。
  5. 文档命令改为显式 `brew install linhay/tap/ski`，避免命中 Homebrew Core 同名包。
- 验证：
  1. `gh release view 2.0.0 --repo linhay/SKIntelligence --json assets` 可见四个 `ski-macos-*` 资产。
  2. `swift test --filter CLIDistributionScriptsTests` 9/9 通过。
